동기(synchronous)
- 호출자(caller)가 피호출자(callee)를 기다림
- 앞의 작업이 끝나야 다음 작업 시작
-> 기다리는 동안 다른 작업을 하지 못해 컴퓨팅 자원 낭비(i/o 작업에 비효율적)
-> I/O작업(프로그램이 cpu 계산이 아니라, 외부와 데이터를 주고받는 작업[네트워크, 파일 시스템, 데이터 베이스등])

[동기 함수]
1. 함수 정의
2. 함수 호출
3. 함수 실행
----------------------------------------------------------------------------------------------------------------------------------------------------------------
비동기(async)
- 기다리는 동안 다른 작업 수행
- 동시에 여러개 처리하는 것처럼 보임
- i/o 작업으로 인한 대기 시간이 많이 발생하고, 그동안 다른 일을 처리해야할 때
(다수가 동시사용하는 웹 서버-db(sqlalchemy), 외부 api 호출(asyncopenai), 파일(aiofiles) i/o로 인해 대기 시간이 자주 발생)


asyncio
: 파이썬에서 비동기 프로그래밍을 지원하는 공식 라이브러리
- async + I/O
- I/O작업을 기다리는 동안 다른 작업을 비동기적으로 처리하게 해줌


코루틴(coroutine)
: 중간에 멈췄다가 다시 이어서 실행할 수 있는 함수(서로 협력하면서 실행되는 함수들)

[비동기식(코루틴 함수)]
1. 함수 정의
2. 함수 호출
3. 코루틴 객체
4. 코루틴 객체를 실행

    await 
    - 여기서 멈추고 다른 코루틴에게 실행을 넘겨
    : 결과가 바로 안나오는 작업을 만나면 그 자리를 양보하고 다른 코루틴을 실행한 다음 나중에 이어서 실행
        1. 비동기 함수 안에서만 사용 가능
        2. 일반 동기함수 앞에는 붙일 수 없음
        3. 기다려야하는 작업일때(awaitavle): 기다려야하는 i/o 작업이며, 비동기버전 함수인 경우 

    🚨 주의점
    - 이벤트 루프 블로킹하면 안됨
    블로킹: 오래걸리는 작업이 실행흐름을 붙잡고 다른 작업에게 제어권을 넘기지 않는것

FastAPI 내부 동작
- 동기 엔드포인트(def)를 작성하면, 이를 이벤트 루프에서 직접 실행하지 않고 스레드풀(Thread Pool)에서 진행
- 실행 결과만 이벤트 루프로 다시 가져와서 응답 처리함

    thread pool
    - 미리 만들어둔 여러개의 스레드 작업자들을 모아둔 풀
    - 이벤트 루프를 막지 않기 위해 동기 작업을 대신 처리하는 별도의 스레드 모음
     🚨 주의사항
    
1) 동기 api를 만들때는, 스레드 풀 개수를 늘려라(40개)
2) 비동기 api를 만들고 싶으면, 코루틴 함수의 내용도 비동기로 만들어라
    - 비동기함수를 정의하고, 안에서 동기식 코드를 작성하면 안된다
3) 비동기 api를 만들고 싶은데, 동기식 함수를 실행해야한다면 run_in_threadpool()사용
